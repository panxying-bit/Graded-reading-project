# PRD：阅读生成结果在网页中的呈现

## 目标

- 将模型返回的英文阅读材料以**可阅读、易扫读**的方式展示，避免「像终端或代码块」的体验。
- 对 **JSON 型绘本结构**（当前为 `level3`）与 **纯文本**（`level1` / `level2`）采用不同但一致的阅读视觉层次。

## 非目标

- 不要求在前端实现富文本编辑或教案排版导出（可后续另立 PRD）。
- 不强制改后端 API 形态；以现有 `text` 字符串为输入，在客户端做解析与展示。

## 内容形态与展示规则

| 来源/形态 | 识别方式 | 展示要求 |
|-----------|----------|----------|
| 纯英文正文 | 非合法 JSON 绘本结构 | 正文区域：适中间距、合宜行长、换行与长单词断行、易读衬线/无衬线字体，避免等宽体占满屏像代码。 |
| Level3 多行正文 | 每行一句的纯英文（无 JSON） | 与 level1/2 相同，用 `pre-wrap` 阅读样式；**不再**使用绘本 JSON 卡片。 |
| 旧版绘本 JSON（若未来其他级别使用） | `title` + `pages[]` 等 | 若解析成功则分「页」卡片展示。 |
| 模型多包了 markdown 代码块 | 内容以 ` ```json ` 包裹 | 在解析前**去掉**围栏，再按上表处理。 |

## 验收标准

1. 选择 `level1` / `level2` 时，结果区**不像**代码块仓库；以阅读正文方式呈现。  
2. `level3` 当前为**每行一句的纯英文**；若某次仍返回可解析的 JSON 绘本，则仍按绘本视图展示。  
3. 若内容无法解析为 JSON 但看起来像正文，**勿报错**，回退为纯文本展示。  
4. **复制**时：绘本复制为可粘贴的纯文本（标题 + 每页标号 + 文本）；纯文本为原文。  
5. 生成结果标题附近展示**英文总词数**（与正文同口径：纯文本为整段按空白分词；绘本为各 `pages[].text` 拼接后计词），便于与「约词数」设定对照。

## 版本与实现对应

- 与 `server/config/levels.yaml` 中 `level3` 的 JSON 约定对齐；若日后扩展其他级别的结构化输出，在本 PRD 增补行并复用「解析 + 按形态渲染」模式。
