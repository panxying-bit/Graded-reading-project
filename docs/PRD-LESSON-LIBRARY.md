# PRD：按级别的课文槽位与本地文库

## 需求解析

- 每个**阅读级别**有固定数量的课文位（你当前约定为 **144 课**，与线下教材 / lesson 表一致）。  
- 每生成一篇，应对应 **一个 lesson 序号**（1…N），便于按课推进、不串课。  
- 用户需要 **一眼总览** 该级别下哪些课已有生成、当前正在编辑第几课，并能在已生成的课文间 **快速切换、复读**，而不是每次只看到「刚刚生成这一篇」。

## 与「先改 PRD 再改代码」

- 多人协作、或要和你手里的 144 篇主题表 / 教研流程对齐时：先把 **N、命名规则、是否允许覆盖、是否上云** 写进 PRD 或一页规格，再动代码。  
- 你单人迭代、范围明确时：可直接实现一版，再用本文档**固化**约定。本文档即对当前实现的说明。

## 信息架构（对用户的友好点）

1. **进度与定位**  
   - 顶部有「**已保存 x / N 课**」，回答「这一级写到哪了」。  
   - 「**当前第 a / N 课** + 上/下」避免只靠记忆这是第几篇。  

2. **一屏总览 144 槽**  
   - **12 列格网**（窄屏 6 列）灰格=未存、绿格=已存、描边=当前课；**悬停可看提示**（含约词数）。点格即**切换**到该课并加载本机已保存内容。  

3. **生成与覆盖**  
   - 显式说明「**保存到第 a 课**」「会**覆盖**本课已保存内容」，减少误伤。  
   - 生成长请求期间若已切换到别的课，**不会**用旧请求结果覆盖当前所见的课文（以 **ref 对齐**当前课号）。

## 数据与存储

- **现版本**：`localStorage` 键 `graded-reading.lessonLibrary.v1`，按 `levelId` 与课序号 **1..N** 存正文与词数等；**不经过后端**（适合单机老师备课；多设备/协作需以后接账号与服务器）。  
- **N 的来源**：`server/config/levels.yaml` 中 `defaults.lessonsPerLevel`（可单级覆盖 `lessonsPerLevel`），`GET /api/levels` 下发明文。

## 与课纲主题表（level3 已实现）

- `server/config/lessons/level3.json`：144 行 `lesson` + `theme`（28 个主题按环状循环，相对均匀）；`level3.csv` 便于 Excel 打开。
- 前端在 **level3** 下根据当前课次请求 `GET /api/levels/level3/lessons`，**自动将「主题」填入** topic，并在课文进度区展示「本课课纲主题」；其他级别无该文件时行为与从前相同。
- 为 **level1 / level2** 做同样能力时，复制为 `level1.json` 等并保证 `lesson` 行数与 `lessonsPerLevel` 一致即可。
